# Week 1 分知识点小习题

请先手算，再用你自己的 NumPy 实现核对；不要查 Lab 或参考实现。

## 1. 张量与 Q/K/V 投影（A）

1. 令 `X` 为 `(3, 4)`，`W_Q`、`W_K`、`W_V` 都为 `(4, 2)`。写出 Q、K、V、`QK^T`、weights 和输出的 shape。
2. 若 `X` 为 `(n, d_model)`，而 `W_Q` 为 `(d_model, d_k)`，证明 Q 的每一行仍对应原序列的一个 token。
3. 构造一个错误的 `W_K` shape，使矩阵乘法报错；写出错误发生在哪两个维度。

## 2. Softmax 与数值稳定性（A/B）

1. 对 `[2, 1, 0]` 手算 softmax，验证和为 1。
2. 将每个数同时加 100，证明 softmax 输出不变。
3. 对 `[1000, 999, 998]` 解释直接计算 `exp` 的风险；写出不溢出的等价计算顺序。
4. 对二维 score 矩阵说明为什么 attention 必须沿最后一维归一化；若沿 `axis=0`，谁在竞争概率？

## 3. Scaled dot-product attention（A/B）

1. 给定两个二维 query 和两个二维 key，手算一个 `QK^T` 矩阵。
2. 设 `d_k=64`，解释除以 `sqrt(d_k)` 前后 score 的量级差异和 softmax 饱和风险。
3. 若两个 key 完全相同，而 value 不同，weights 和输出各会保留什么区别？
4. 构造一个 token 更偏向读取自己的两-token 例子；解释 Q、K、V 分别应具备的关系。

## 4. 诊断题（C）

1. 设计一个最小输入，能抓住“实现遗漏缩放项”的错误；说明预期观察什么性质。
2. 设计一个最小输入，能抓住“softmax 沿错轴”的错误；不得只写文字。
3. 某实现输出 shape 正确但所有 query 得到相同 weights。列出三个不同的可能原因，并给出各自的验证办法。
