# Week 0：先看懂信息怎样流动

> 本周采用课程统一的 [七步闭环](../../docs/weekly-module-template.md)：导言 → 材料 → 摘抄笔记 → Lab → Lab feedback → 自主 demo → 工程 feedback。以下任务 0.1–0.5 是第 4 步 Lab 之前的前置理解链。

## 这一周在整门课里做什么？

下一周你会让一个 token 根据内容去“读取”其他 token。那时你会看到 Q、K、V、矩阵乘法和 softmax。若现在只把这些名字背下来，写 Lab 时很容易陷入“报错就改 shape”的猜测。

所以 Week 0 不要求你预先学完一整套数学。我们只做一件事：拿一组极小的 token 向量，准确追踪它每经过一步后**变成了什么、shape 是什么、数值为什么会变化**。完成后，Q/K/V 不再是新术语，而只是“同一组 token 的三种投影”。

## 本周结束时，你应该能做什么？

给你 3 个 token、每个 token 2 个数，以及一个小矩阵时，你能：

1. 说明每一行代表一个 token、每一列代表一个特征；
2. 判断两矩阵能否相乘，并预测结果 shape；
3. 计算两个 token 的点积，并解释它为什么可以当作匹配分数；
4. 将一行分数转换为和为 1 的权重，并解释为什么要先做数值稳定处理；
5. 说出这些步骤将在 Week 1 的 attention 计算里各出现一次。

这五项不是额外负担，而是 Week 1 的直接入口：若第 2 项不稳，就回到矩阵乘法；若第 4 项不稳，就回到 softmax。不要带着模糊理解进入 Q/K/V。

## 任务链：从可见例子到 Week 1

### 任务 0.1：用一个小例子建立“token 是行”的直觉

从三个二维 token 开始，例如“每行一个 token、两列两个特征”。不用任何 Transformer 名词，只回答：改变某一行会影响哪一个 token？改变某一列又意味着什么？

**完成证据：** 你能画出一个 `(3, 2)` 表，并用一句话解释行和列。

### 任务 0.2：让信息通过一个矩阵

把 `(3, 2)` 的 token 表乘以 `(2, 2)` 的权重矩阵。先在纸上检查内维度，再计算一个元素，最后预测完整输出的 shape。

**为什么现在做：** Week 1 的 Q、K、V 都是这个操作重复三次。

**卡住时回退：** 如果不确定能否相乘，只检查左矩阵列数是否等于右矩阵行数；不要先运行代码猜答案。

### 任务 0.3：把“匹配程度”变成一个数字

对两个二维 token 做点积。改变其中一个 token 的一个数，再比较点积如何变化。这里要建立的是：dot product 是一种可计算的匹配分数，不是“语义相似度”的万能定义。

**为什么现在做：** Week 1 会把每个 query 与每个 key 的点积放进一个矩阵。

### 任务 0.4：把分数变成可分配的权重

从一行三个分数开始，手算 softmax。再把三个分数都加上一个很大的常数，观察理论上输出应保持不变。最后思考：如果直接计算很大的指数，会发生什么？

**为什么现在做：** attention 不直接使用 dot-product 分数，而是先把它们变成每行和为 1 的读取权重。

### 任务 0.5：预演 Week 1 的完整信息流

不实现 attention。只在纸上画出：`token 表 → 三次矩阵投影 → 两两匹配分数 → 每行权重 → 加权读取结果`。为每一个箭头标出你已在本周完成的前置任务编号。

**通过门槛：** 不看材料，能讲清这张图，并指出 Week 1 唯一新增的部分是“把这些熟悉步骤连起来”。

## 本周学习材料与练习

- 只阅读与你正在做的任务对应的线性代数、softmax 和 attention 预备内容；不要试图通读整本 D2L。
- 每个任务都先手算一个极小例子，再用 NumPy 验证。
- 每完成一个任务，写下“这个操作在 Week 1 的哪一步会再次出现”。
