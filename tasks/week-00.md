# Week 0 任务：从 token 表走到一次矩阵投影

这些任务按顺序完成。每一步只为下一步增加一个必要动作，并对应 Week 0 Lab 的一个关卡。

1. **把 token 放入 NumPy 数组。** 创建一个小型二维 `np.ndarray` 作为 token 表，例如三行 token、两列特征。用自己的话标注行、列和它的 shape。对应 Lab 关卡 1–2。
2. **给投影选一个矩阵。** 创建一个与 token 表兼容、且不是单位矩阵的二维投影矩阵。预测 `tokens × projection` 的输出 shape；只判断 shape，暂不计算结果。对应 Lab 关卡 3。
3. **手算一个位置。** 选定输出的一个行列位置，写出它取自 token 表的哪一行和投影矩阵的哪一列；再手算这个点积。对应 Lab 关卡 4。
4. **组装整张投影表。** 使用 Lab 的手写矩阵乘法函数，得到完整投影结果；核对每一行仍对应原来的 token。对应 Lab 关卡 5。
5. **让 NumPy 成为对照，而不是替代思考。** 用 `@` 计算同一个乘积，确认它与手写结果一致。记录一个相同点与一个你原先容易弄反的维度。对应 mini demo。
6. **制造一个有意义的错误。** 让两个矩阵的内维度故意不匹配，调用诊断函数；解释它为什么不能相乘，以及这个错误若发生在 \(Q=XW_Q\) 时意味着什么。对应 Lab 关卡 6。

**进入概念作业的门槛：** 不看代码，能画出一次 `token 表 → 投影矩阵 → 新 token 表` 的输入、输出及三者 shape；并能解释一个输出位置的来源。
