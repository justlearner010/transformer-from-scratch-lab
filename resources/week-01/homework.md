# Week 1 Lab 后工程作业

请在自己的 `answers/week-01.md` 中完成。每一题必须引用你的 Lab 输入、
grader 类别、断言或 demo 观察；不得仅复述教材。

1. **完整计算。** 自己构造三 token、二维 Q/K/V，完整计算一个 query 的
   score、scaled score、权重和输出。解释每一步的 shape。
2. **设计理由。** 为什么 stable softmax 必须先减去每行最大值，且必须沿
   最后一维做？用你自己的极端数值或 axis 反例支持。
3. **边界诊断。** 选择一个 Q/K 特征维度不匹配或 key/value token 数不匹配
   的输入。它应在哪个函数被拒绝？若拖到后面才失败，排查成本为何更高？
4. **机制错误。** 给出一个输出 shape 正确、但机制错误的实现反例。它违反
   score、缩放、softmax 或 V 读取中的哪一条契约？
5. **失败反馈。** 选一次真实或主动注入的错误，写出：现象、错误类别、
   原因假设、最小验证、修复后成立的性质和预防策略。
6. **迁移取舍。** 你的 independent inspector 与 Lab 如何分工？为何它应
   调用公开接口而不复制 attention 内部实现？
7. **全局解释。** 用 250–350 字回答：“attention 为什么是内容相关的读取，
   而不只是平均池化？”必须引用一个自己构造的数值例子。

## 提交标准

- 第 1、2、3、4 题必须展示计算、输入或断言证据；
- 第 5 题必须是真实或主动注入的失败，不得虚构 grader 输出；
- 第 7 题必须区分 score、weight 与 value；
- 未关闭的 P0 错误会阻止进入 causal mask。
