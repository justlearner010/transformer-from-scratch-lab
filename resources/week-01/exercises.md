# Gate 1–4：概念作业（先于 Lab）

请先手算、预测和解释；不要查看 `labs/week-01/src/attention.py` 的实现答案或隐藏
评分样例。每个 gate 达到进入条件后再继续。

## Gate 1（P1）：Q/K/V 投影与角色

1. 令 `X` 为 `(3,4)`，三个权重为 `(4,2)`。写出 Q、K、V 的 shape，并
   标注每一维的语义。
2. 选一个非单位投影矩阵，手算 `Q=XW_Q` 的一个输出位置。解释它为何仍
   属于原 token 的一行表示。
3. 不运行代码，预测：只改变 `W_V` 时，哪些量必须不变，哪些量可能改变？

**进入 Gate 2：** 能说明 Q 负责“问什么”、K 负责“与什么比较”、V 负责
“读什么”，而不是把三者描述为三份固定 embedding。

## Gate 2（P0）：score 的行列语义

1. 用两个 query 和三个 key 手算 `Q @ K.T`，并写出每个位置 `scores[i,j]`
   的含义。
2. 对其中一行，先预测哪个 key 的分数最大，再计算验证。若预测错，说明
   是哪一个向量关系被误判。
3. 构造一个 `Q @ K` 不可计算或语义错误的例子，解释为什么转置必须发生
   在 K 上。

**进入 Gate 3：** 能在不看公式的情况下说明“同一行是谁在竞争”。

## Gate 3（P0/P1）：缩放与稳定 softmax

1. 证明对任意常数 `c`，`softmax(z)=softmax(z-c)`；写出稳定计算顺序。
2. 对 `[1000,999,998]` 说明直接 `exp` 的风险，并预测稳定实现应保留的
   相对大小。
3. 对一个 `(2,3)` score 矩阵，分别说明 `axis=-1` 与 `axis=0` 时谁在
   竞争概率；构造能区分二者的具体数组。
4. 假设 Q、K 元素均值为 0、方差为 1，说明 dot product 的方差为什么随
   `d_k` 增长，以及为何不能用 token 数替代 `d_k`。

**进入 Gate 4：** 能手算一行权重、说明每行和为 1，并给出一个 axis 或
数值稳定性失败的最小反例。

## Gate 4（P0）：从 weights 读取 V

1. 设 weights 为 `(n_query,n_key)`、V 为 `(n_key,d_v)`，写出输出 shape
   并解释为什么输出的特征宽度来自 V。
2. 保持 Q/K 不变，只改变一个 V 行；先预测哪些 weights 与 outputs 会变，
   再用一个小数值例子验证。
3. 写出完整数据流：`X → Q/K/V → scores → scaled scores → weights → output`。
   在每条箭头旁标出 shape。

**进入 Lab：** 完成上述数据流而没有 shape 或因果错误；若错误，回到相应
gate，而不是靠运行结果猜修复。
