# Agent Decision Contract

## 目的

本契约让 AI 做真实的学习诊断与路径推荐，同时不让模型自行改变课程标准、
证据要求或学习状态。课程规则必须先于模型调用存在，并且可版本化、测试和回滚。

## 职权边界

| 角色 | 可以决定 | 不能决定 |
| --- | --- | --- |
| 课程负责人 | 知识图、节点重要性、目标深度、证据与发布版本 | 依据单个学生答案临时降低标准 |
| AI 诊断器 | 在 Gate 允许的诊断标签中选择候选、引用证据、推荐允许的补练路径，或弃权 | 新增标签、修改节点优先级、直接 PASS、跳过 Gate |
| Policy 与状态机 | 只执行契约允许且证据充分的状态变化 | 解释答案、创造课程任务 |
| 学生 | 作答、运行实验、提交证据 | 修改课程判定规则 |

## 两种层级不能混淆

课程层级是已发布规则：`primary_type`、`importance`、`dependency_role`、
`target_depth`。学生层级是动态证据状态：某学生对该节点目前达到的深度，以及
证据是否不足。AI 只判断后者，不能在运行时改前者。

## AI 的受限决策协议

每个可评估 Gate 必须声明：

1. 可接受的证据 criterion；
2. 可选诊断标签与每个标签允许的路径；
3. `unknown` / `insufficient_evidence` 的弃权路径；
4. PASS 所需的独立证据与确定性检查。

AI 的结果只能是“带答案原文引证的候选诊断”。没有引证、诊断不在白名单、
多个路径冲突或证据不足时，Policy 必须停在 `evidence_pending` 或 `escalated`，
而不是猜测通过。

## 风险分级

| 决策 | AI 可独立输出 | 最终执行条件 |
| --- | --- | --- |
| 解释或提示 | 是 | 不改变状态 |
| 推荐下一道补练 | 是，但仅限 manifest 白名单 | Policy 校验标签和路径 |
| 错误诊断 | 是，但必须引用证据或弃权 | 可用最小区分实验复核 |
| PASS / 解锁 | 否 | rubric、提交证据、Policy 和状态机共同满足 |
| 修改课程结构 | 否 | 课程负责人审核、版本发布 |

## Week 1 试点验收

- 每个 Gate 模板只要求其证据类型需要的栏目；
- 同一份提交和同一 rubric / verifier 版本复用同一判定；
- 非法诊断、无引证诊断和状态字段注入均不能改变状态；
- 学生可在自己的 `main` 完成作答，无需理解分支；
- 完成 Gate 0 后，由真实学习证据而不是模型措辞决定下一步。
