# 学习入口

你只需要关心四件事：**读什么、做什么、写在哪里、下一步是什么**。

> 课程的维护文件仍在仓库里，但 VS Code 默认不会显示它们。它们负责评分、状态与课程规则，不是学习时需要翻看的内容。

评分器、运行时和课程配置等维护实现会被隐藏。

| 阶段 | 导航与材料 | 实践 | 你的学习证据 |

## 当前学习：Week 1

Week 1 的问题是：**一个 token 如何依据内容选择性读取其他 token？**

1. 先启动学习 Agent：

   ```bash
   uv run learning-os agent week-01
   ```

2. Agent 会告诉你当前 Gate、要完成的任务、对应作答文件和提交检查点。
3. 按下方“Gate 导航”阅读材料或完成 Lab。
4. 只填写 Agent 指定的一个文件：`homework_answer/week-01/gate-xx.md`。
5. 保存后，亲手提交本次作答：

   ```bash
   git add homework_answer/week-01/gate-xx.md
   git commit -m "answer: week 01 gate xx"
   ```

6. 回到 Agent，输入 `/submit`。它只判断证据与给出下一步提示；不会替你写答案、提交代码或推送仓库。

## 你在文件夹中会看到什么

| 文件夹 | 你用它做什么 |
| --- | --- |
| [`weeks/`](weeks/) | 了解每周为什么学、完成后能做什么。 |
| [`resources/`](resources/) | 阅读材料、练习、作业说明。 |

| `homework_answer/` | **你的学习证据区**；按 Week 保存笔记、作答与 Gate 证据。Week 1 首次启动 Agent 后生成。 |

## 常用 Agent 命令

| 命令 | 用途 |
| --- | --- |
| `/status` | 查看现在在哪个 Gate、是否能继续。 |
| `/submit` | 在你手动 commit 后，提交当前作答证据。 |
| `/retry` | 已配置或修复判定环境后，重新运行判定。 |
| `/revise` | Agent 要求补强或诊断后，开启当前 Gate 的修订；修改并 commit 后再 `/submit`。 |

如果 VS Code 需要查看维护文件：在资源管理器的“…”菜单中选择 **Show Excluded Files**，或临时修改 [`.vscode/settings.json`](.vscode/settings.json)。学习时不需要这样做。

## 还没有准备环境？

